# 智能家居物联网项目（STM32 + FreeRTOS + OneNET + uni-app）

## 1. 项目简介（简历可用）

这是一个基于 `STM32F103C8` 的智能家居监测与控制系统，包含**设备端**（传感器采集、执行控制、RTOS任务调度、云端上报）和**移动端**（uni-app 可视化与远程控制）。

项目主要实现了：

- 多传感器实时采集：温湿度（AHT20）、光照（BH1750）、甲烷（MQ-4）、一氧化碳（MQ-7）、人体红外（HC-SR501）
- 本地联动控制：人体检测触发台灯（LED）开关
- 云端数据上报：通过 ESP8266 接入 OneNET，使用 MQTT 上报物模型数据
- 云端反控下发：订阅属性设置主题，实现手机端远程控制 LED
- 本地显示：LCD 实时展示传感器状态、WiFi/MQTT 连接状态

## 2. 技术栈总览（按层拆解）

### 2.1 硬件与芯片

- 主控：STM32F103C8（ARM Cortex-M3）
- 通信模组：ESP8266-01S（AT 指令透传 + MQTT）
- 传感器：
	- AHT20（温湿度）
	- BH1750（光照）
	- MQ-4（甲烷，DO）
	- MQ-7（一氧化碳，DO）
	- HC-SR501（人体红外）
- 执行器：LED（台灯）
- 显示：TFT LCD

### 2.2 嵌入式软件

- 语言：C
- RTOS：FreeRTOS
	- 抢占式调度
	- 多任务并发
	- 队列通信（`xQueueCreate / xQueueOverwrite / xQueueReceive`）
- 芯片库：STM32 StdPeriph Library（GPIO/UART/I2C/SPI/TIM 等）
- 驱动开发：
	- 软件 I2C（PB6/PB7，AHT20 与 BH1750 共线）
	- UART1 调试串口
	- UART2 与 ESP8266 通信

### 2.3 网络与物联网协议

- WiFi 接入：ESP8266 `AT+CWJAP`
- 云平台：OneNET（中国移动物联网平台）
- 协议：MQTT（1883）
- 主题：
	- 上报：`$sys/<product>/<device>/thing/property/post`
	- 订阅：`$sys/<product>/<device>/thing/property/set`
- 数据格式：JSON 物模型属性（temp/humi/lux/LED/CO/CH4/human）

### 2.4 前端/移动端

- 框架：uni-app + Vue2
- 页面：单页仪表盘（温湿度/气体/光照/人体/LED）
- API：OneNET HTTP OpenAPI
	- 查询属性：`thingmodel/query-device-property`
	- 设置属性：`thingmodel/set-device-property`
- 鉴权：HMAC-SHA1 Token（JS 端生成）

### 2.5 开发工具链

- IDE：Keil uVision
- 工程：`Project.uvprojx`
- 工具链：ARMCC5（RVDS）
- 目标芯片包：`Keil.STM32F1xx_DFP`
- 调试：串口日志 + LCD 状态可视化


## 4. 关键实现细节（面试最容易加分）

### 4.1 FreeRTOS 多任务职责拆分

- `task2`：读取 AHT20/BH1750，刷新 LCD，写入环境队列
- `task3`：读取 MQ4/MQ7 数字量，写入气体队列
- `task4`：读取人体红外，执行本地/云端优先级控制，写入状态队列
- `task5`：
	- 负责 WiFi 连接与 MQTT 连接管理
	- 订阅云端控制主题
	- 解析下行控制指令（LED true/false）
	- 组包上报 JSON 物模型
	- 保活心跳与断链重连

### 4.2 队列与数据快照

- 使用 3 个队列（环境/气体/状态）进行任务间解耦
- 使用 `xQueueOverwrite` 保留“最新值”，降低堆积延迟
- 网络任务按“快照”组包，避免多源数据不同步

### 4.3 云控优先级策略

- 默认策略：人体感应控制 LED（有人亮灯，无人灭灯）
- 云控策略：收到云端 LED 指令后进入云控模式
- 超时回退：30 秒未收到新云控命令，自动回到本地联动

### 4.4 MQTT 兼容策略（很有亮点）

- 优先走 ESP8266 内置 MQTT AT 指令
- 当长用户名/密码等场景 AT 能力不足时，自动回退到“原始 TCP + 手动 MQTT 报文”
- 已实现手动封包：`CONNECT / SUBSCRIBE / PUBLISH / PING / DISCONNECT`
- 已实现 `Remaining Length` 编码解析与 `SUBACK/PUBACK` 处理

### 4.5 连接稳定性设计

- 轮询检测连接状态：`CLOSED / WIFI DISCONNECT` 等关键字
- MQTT 心跳：定时 PING 保活
- 发送失败计数与重连触发（连续失败阈值）
- 接收缓冲保护：接近 3/4 容量时主动清理，防止串口缓存撑爆

### 4.6 传感器驱动实现

- AHT20/BH1750 基于软件 I2C 实现
- AHT20 按 20-bit 原始数据计算温湿度
- BH1750 按原始值换算 lux（`raw / 1.2`）
- MQ4/MQ7 使用 DO 数字量阈值输入

## 10. 你在面试中可用的 60 秒口述模板

我做了一个基于 STM32F103 的智能家居物联网项目，底层用 FreeRTOS 把采集、控制、联网拆成多个任务。设备端采集 AHT20、BH1750、MQ4/MQ7 和人体红外数据，本地通过 LCD 展示状态，并根据人体检测自动控制 LED。联网方面通过 ESP8266 接入 OneNET，用 MQTT 做属性上报和云端下发控制。我还做了链路稳定性设计，包括心跳保活、断链重连、发布失败重试，以及 AT-MQTT 不兼容时回退到原始 TCP 手动封包。前端用 uni-app 做了可视化和远程开关控制，形成完整闭环。
